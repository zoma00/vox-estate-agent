After merging a pull request on GitHub, your remote `main` branch is updated, but your local `main` branch remains unchanged. Here's why running these commands is important:

1. **`git checkout main`**:
   - Switches you from your feature branch (`feature/speech-recognition`) to the `main` branch.
   - This is necessary because you can't update a branch you're currently on.

2. **`git pull origin main`**:
   - Fetches the latest changes from the remote `main` branch and merges them into your local `main` branch.
   - This ensures your local `main` branch matches the remote one, including all merged changes.

### What Happens If You Skip These Steps?
- Your local `main` branch will be out of sync with the remote.
- Future branches created from `main` will be based on an outdated version.
- You might encounter merge conflicts later when trying to push changes.
