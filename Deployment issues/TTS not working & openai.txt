You're seeing that message because you ran SSH from the server shell (the local machine) while giving a client-side private-key path that doesn't exist on that machine. In short: the SSH client is looking for the private key on the computer you ran the ssh command from — not on the remote host — and it couldn't find it, so no key was offered and the server rejected you.

Quick explanation
- Warning: Identity file /home/deploy/.ssh/id_ed25519 not accessible: No such file or directory
  - Means the SSH client tried to load /home/deploy/.ssh/id_ed25519 on the *local* machine and it doesn't exist there.
- Permission denied (publickey)
  - The server refused the connection because no acceptable publickey was presented.

Two common situations that cause this:
- You intended to run the SSH command from your laptop but accidentally ran it from the server shell. The private key lives on your laptop (e.g. ~/.ssh/id_ed25519) so it can't be read from the server.
- You really intended to SSH from the server to the server (loopback), but you don't have the private key on the server at the specified path.
